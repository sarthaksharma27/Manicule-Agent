# Upload API

## Overview

The Upload API allows clients to upload files to the server. This document outlines the available endpoints and their usage.

## Endpoints

### POST /upload/v2

This endpoint is used to upload files to the server. It replaces the deprecated `/upload` endpoint.

<ParamField name="file" type="File" required={true}>
  The file to be uploaded. Must be included in the request body as form-data.
</ParamField>

### Example Request

```http
POST /upload/v2 HTTP/1.1
Host: api.example.com
Content-Type: multipart/form-data; boundary=---011000010111000001101001

---011000010111000001101001
Content-Disposition: form-data; name="file"; filename="example.txt"
Content-Type: text/plain

(file content here)
---011000010111000001101001--
```

### Example Response

```json
{
  "status": "success",
  "message": "File uploaded successfully",
  "fileUrl": "https://api.example.com/uploads/example.txt"
}
```

## Notes

- Ensure that the file is sent as form-data.
- The maximum file size allowed is 10MB.
- Supported file types include `.txt`, `.jpg`, `.png`, and `.pdf`.

## Deprecation Notice

The `/upload` endpoint has been deprecated and replaced by `/upload/v2`. Please update your applications to use the new endpoint to ensure continued functionality.
