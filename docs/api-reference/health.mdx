# File Upload API

The File Upload API allows clients to upload files to the server. This document provides details on the available endpoints, request parameters, and response formats.

## Endpoints

### POST /upload/v2

<Warning>
This endpoint replaces the deprecated `/upload` endpoint. Ensure your integrations are updated to use `/upload/v2`.
</Warning>

Uploads a single file to the server.

#### Request

- **URL**: `/upload/v2`
- **Method**: `POST`
- **Headers**: 
  - `Content-Type: multipart/form-data`
- **Body**: 
  - Form data with a single file field named `file`.

<ParamField path="file" type="file" required>
The file to be uploaded. Must be included in the form data.
</ParamField>

#### Response

- **Status Code**: `200 OK`
  - <ResponseField name="message" type="string">Confirmation message indicating successful upload.</ResponseField>

- **Status Code**: `400 Bad Request`
  - <ResponseField name="error" type="string">Error message indicating no file was uploaded.</ResponseField>

#### Example Request

```bash
curl -X POST http://localhost:4000/upload/v2 \
  -H "Content-Type: multipart/form-data" \
  -F "file=@/path/to/your/file.txt"
```

#### Example Response

**Success (200 OK)**

```json
{
  "message": "File received"
}
```

**Error (400 Bad Request)**

```json
{
  "error": "No file uploaded"
}
```

## Setup

Ensure your environment meets the following requirements:

- **Node.js**: Version `>=18.12`
- **Dependencies**: 
  - `express`: `^5.2.1`
  - `multer`: `^2.0.2`
  - `cors`: `^2.8.5`

## Development

To start the development server, use the following command:

```bash
npm run dev
```

This will start the server with `nodemon` and `ts-node`, watching for changes in the `src` directory.

## Build

To compile the TypeScript code to JavaScript, run:

```bash
npm run build
```
